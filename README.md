# Xiangqi-SwiftUI
Study SwiftUI, try to make a Xiangqi game

### current progress: 
- board
- pieces
- game manager

### next step: 
- move validation
- check & check mate
- display pieces
- display available moves
- surrender, asking for draw
- move rules (continuos check, continuos capture...)
- timer

### future plan:
- screen transitioning
- invalid move toast message
- view game log
- undo moves
- blind chess
- setting (time, background, pieces...)
- playing via WebRTC / socket
